# HISTORY

Evolution of the GitCube Lab project.

This file explains how the ideas developed.

---

# Phase 1 – Graph School

Initial idea:

Teach architectural reasoning through graph grammars.

Policies defined allowed structural relations between layers.

Examples:

layered  
clean/onion  
hexagonal  
microservices  
event-driven

This stage focused on learning architecture patterns.

---

# Phase 2 – GraphEval

Next step was creating a richer structural risk engine.

GraphEval introduced:

cycle detection  
layer violations  
density analysis  
risk scoring  
DNA signatures

This became the core structural evaluator.

---

# Phase 3 – Agent repair loops

The system evolved to include agents that attempt to repair
bad architectures.

Agents perform:

mutation  
evaluation  
iteration

Goal:

improve architecture automatically.

---

# Phase 4 – Structural memory

Memory modules were introduced to store evaluation traces.

Important concept:

architecture atoms

Later extension:

flower invariant

Flower measures trajectory shape in risk-entropy space.

---

# Phase 5 – Graph AI

Bridge between LLMs and architecture reasoning.

Pipeline:

prompt  
→ graph generation  
→ evaluation  
→ auto repair

This allows language models to produce candidate architectures.

---

# Future direction

Human-AI symbiosis for architectural reasoning.

Human provides intent.

System evaluates structure,
suggests repairs,
and accumulates long-term knowledge.
